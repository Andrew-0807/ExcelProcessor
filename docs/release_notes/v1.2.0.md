# Release v1.2.0 - Standalone App with Auto-Updates

## Overview
This release introduces the standalone executable version of Excel Processor with automatic update capabilities. The application now runs entirely on the client's machine without requiring server hosting.

## What's New

### Standalone Executable
- **Single-file executable**: No installation required, just download and run
- **All processing modules bundled**: Borderou, CardCec, and Classes modules included
- **System tray application**: Easy access from the system tray with right-click menu
- **Auto-update system**: Automatically checks for and installs updates from GitHub Releases

### Auto-Update Features
- **Automatic update checking**: Checks for updates every hour (configurable)
- **Manual update check**: Option to manually check for updates from the system tray
- **Seamless updates**: Downloads and installs updates automatically
- **Version tracking**: Compares local version with latest GitHub release

### Improvements
- **Better error handling**: Improved error messages and user notifications
- **Progress tracking**: Update download progress is displayed to the user
- **Restart handling**: Application restarts automatically after update installation

## Installation

### For End Users
1. Download `ExcelProcessor.exe` from the [latest release](https://github.com/Andrew-0807/ExcelProcessor/releases/latest)
2. Place the executable anywhere on your computer
3. Double-click to run the application
4. The app will open in your browser at `http://127.0.0.1:5000`

### System Requirements
- Windows 10 or later
- No additional dependencies required (all bundled)
- Internet connection for auto-updates (optional)

## Usage

### Starting the Application
- **Windows**: Double-click `ExcelProcessor.exe`
- The app will:
  - Start a local web server
  - Open your default browser
  - Show a system tray icon for easy access

### Processing Files
1. Drag and drop Excel files (.xlsx, .xls, .csv) into the upload area
2. Select a processing mode:
   - **Adaos**: Add percentage markup to values
   - **SGR**: Apply SGR processing rules
   - **Minus**: Subtract specified values
   - **Extract**: Extract specific data
   - **Borderou**: Transform borderou format
   - **CardCec**: Process card data
3. Click "Process Files"
4. Download the processed files

### System Tray Features
- **Open Excel Processor**: Opens the web interface
- **Check for updates**: Manually check for new versions
- **Exit**: Close the application

## Updating

### Automatic Updates
- The app automatically checks for updates every hour
- When an update is available, it will:
  - Download the new version
  - Install the update
  - Restart the application
  - Notify you of the update

### Manual Updates
1. Right-click the system tray icon
2. Select "Check for updates"
3. If an update is available, it will be downloaded and installed

## Technical Details

### Build Information
- **Built with**: PyInstaller
- **Python version**: 3.11
- **Update mechanism**: GitHub Releases API
- **Local processing**: All file processing happens on the client machine

### Architecture
```
ExcelProcessor.exe
├── launcher.py          # System tray and update manager
├── server.py            # Flask web server
├── auto_update.py       # Update checking and downloading
├── borderou/            # Borderou processing module
├── CardCec/             # CardCec processing module
├── classes/             # Processing classes
├── templates/           # HTML templates
└── static/              # CSS, JS, and icons
```

## Troubleshooting

### App won't start
- Check Windows Defender or antivirus isn't blocking the executable
- Run as administrator if needed
- Check that port 5000 isn't already in use

### Updates not working
- Ensure you have an internet connection
- Check that GitHub is accessible
- Verify the release tag follows the `vX.Y.Z` format

### Processing errors
- Ensure your files are valid Excel files (.xlsx, .xls, .csv)
- Check that the selected processing mode matches your file type
- Try processing smaller batches of files

## Migration from Previous Versions

If you were running the development version:
1. Download the new `ExcelProcessor.exe`
2. Replace your old executable
3. The app will automatically check for updates going forward
